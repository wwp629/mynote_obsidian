Obsidian 提供了灵活的视图和模式，让您可以在 Markdown 语法的书写效率与最终的阅读体验之间自由切换。核心概念分为两层：**视图**决定了您当前是“编辑”还是“阅读”，而**编辑模式**则决定了您在编辑时的具体样式。

## 视图：编辑与阅读

Obsidian 有两种主要视图，服务于不同的使用场景。

-   **编辑视图 (Editing View)**: 用于创建和修改笔记内容。
-   **阅读视图 (Reading View)**: 用于展示渲染后的最终效果，隐藏所有 Markdown 语法，提供纯净的阅读体验。

### 如何切换视图

-   **快捷键**: 使用 `Ctrl+E` (macOS: `Cmd+E`) 在两种视图间快速切换。
-   **图标点击**: 点击编辑器右上角的书本或铅笔图标。
> [!NOTE] 您需要启用 **设置 → 外观 → 显示标签页标题栏** 才能看到切换图标。
-   **并排预览**: 按住 `Ctrl` (macOS: `Cmd`) 并点击视图切换图标，可以在新标签页中并排打开同一笔记的两种视图，实现实时对照。

## 编辑模式：实时预览与源码模式

在**编辑视图**下，您可以选择两种不同的编辑模式，以适应您的写作偏好。

| 特性       | 实时预览 (Live Preview)                | 源码模式 (Source Mode)              |
| :------- | :--------------------------------- | :------------------------------ |
| **显示效果** | “所见即所得”，实时渲染 Markdown 格式，如加粗、标题等。  | 以纯文本形式完整显示所有 Markdown 语法，不进行渲染。 |
| **语法查看** | 仅当光标移动到对应文本上时，才会临时显示其 Markdown 语法。 | 始终可见。                           |
| **适合场景** | 无缝的写作体验，无需频繁切换到阅读视图即可预览效果。         | 需要精确控制 Markdown 源码，或偏好纯文本写作的用户。 |

> [!TIP] 推荐使用实时预览
> 对于大多数用户而言，**实时预览**模式提供了最佳的平衡，因为它在编辑时就能直观地展示格式效果，极大地减少了切换到阅读视图的必要性。

## 相关设置

您可以在 Obsidian 的设置中自定义默认行为：

-   **默认视图**: **设置 → 编辑器 → 新标签页的默认视图**，可设为“编辑视图”或“阅读视图”。
-   **默认编辑模式**: **设置 → 编辑器 → 默认编辑模式**，可设为“实时预览”或“源码模式”。
