在 Obsidian 中实现多端同步有多种方法，主要分为使用 Git 进行版本控制和使用云盘服务两种。结合两者优点是目前较为推荐的方案。

## 方案一：使用 Git 进行版本控制

通过 Obsidian Git 插件，可以利用 Git 对笔记库进行版本管理和同步。

-   **优点**:
    -   精确的 版本控制，可以追踪每一次修改，轻松回滚。
    -   公开仓库免费，适合纯文本笔记。
-   **缺点**:
    -   初始设置相对复杂。
    -   不适合同步大型附件（图片、PDF），会拖慢速度并迅速消耗 GitHub 等平台的免费空间。

### 1. 初始设置 (命令行)

初次配置建议使用命令行完成，后续可使用插件进行日常同步。

1.  **安装插件**: 在 Obsidian 中安装 `Obsidian Git` 插件，并设置好 `git.exe` 的正确路径。
    ![[Pasted image 20251108212233.png]]
2.  **配置 .gitignore**: 在仓库根目录创建 `.gitignore` 文件，排除无需同步的文件夹，以优化同步效率。
    ```gitignore
    # Obsidian 配置文件，通常设备间独立
    .obsidian/
    
    # 附件目录
    /00 Attachment(附件保存目录)/
    ```
3.  **初始化并推送**:
    ```bash
    git init
    git branch -M main
    git add .
    git commit -m "初始化提交"
    git remote add origin https://github.com/wwp629/mynote_obsidian.git
    git push -u origin main
    ```

### 2. 日常使用 (插件)

-   **推送 (Push)**: 点击左侧面板的 `Commit and push all changes` 按钮即可一键提交与推送。
    ![[Pasted image 20251108213628.png]]
-   **拉取 (Pull)**: 点击 `Pull` 按钮从远程仓库同步更新。

### 3. 注意事项

> [!WARNING] 避免 同步冲突
> 为避免数据冲突，请严格遵守“单向编辑”原则：
> 1.  在设备 A 修改后，立即推送。
> 2.  在设备 B 开始编辑前，务必先拉取最新版本。

## 方案二：使用云盘 (如 OneDrive) 同步

将整个笔记库文件夹放置在 OneDrive、Dropbox 或其他云盘服务的同步目录中。

-   **优点**:
    -   设置简单，自动在后台同步。
    -   无缝同步所有文件，包括大型附件。
-   **缺点**:
    -   版本控制能力弱或缺失。
    -   偶尔可能因同步延迟或错误产生文件冲突。

## 推荐方案：Git + 云盘混合模式

这是兼顾便利性与安全性的最佳实践方案。

1.  **主力同步**: 使用 OneDrive 等云盘作为主力，将整个库（包括附件）置于其同步目录中，实现所有文件的实时自动同步。
2.  **版本备份**: 在此基础上，使用 Git 对笔记库进行版本控制，但务必通过 `.gitignore` 排除附件文件夹。

通过这种方式，OneDrive 保证了日常使用的便捷与附件的无缝同步，而 Git 则作为一道强大的防线，用于在发生文件冲突或内容丢失等意外时进行版本恢复。
